# POG ROLLS

Локальный сайт для настольной игры: профили пяти игроков, казна, игровое поле, учёт игр и будущий магазин.

## Быстрый запуск на другом ПК

1. Скопируйте всю папку проекта на второй ПК.
2. Установите [Node.js LTS](https://nodejs.org/).
3. Откройте папку проекта в PowerShell или терминале.
4. Один раз выполните:

   ```powershell
   npm install
   ```

5. Запустите сервер:

   ```powershell
   node server.js
   ```

   Либо дважды нажмите `start-server.cmd`.

6. Откройте в браузере: `http://127.0.0.1:3000`.

Сервер должен оставаться запущенным, пока вы пользуетесь сайтом. Остановить его можно в окне терминала сочетанием `Ctrl+C`.

## Важные данные

Все балансы, инвентарь и история игр хранятся в файле:

```text
data/db.json
```

Перед переносом проекта или крупными изменениями скопируйте этот файл как резервную копию. Не удаляйте папку `assets`: в ней находятся карты, фоновые изображения, обложки и предметы.

## Страницы

- `index.html` - стартовая страница, выбор профиля, казна и игровое поле.
- `treasury.html` - казна: выдача и списание монет, поиск и назначение текущих игр через HowLongToBeat.
- `player-1.html` ... `player-5.html` - профили игроков.
- `shop.html?player=1` ... `shop.html?player=5` - заготовка отдельной страницы магазина игрока.
- `board.html` - игровое поле с перетаскиваемыми фишками и кликабельными клетками.

## Текущие правила

- У игрока максимум 6 предметов.
- Каждый предмет занимает отдельную ячейку, в том числе одинаковые предметы.
- При полном инвентаре нельзя купить новый предмет или крутить слот-машину.
- У текущей игры есть таймер.
- «Игра пройдена» переносит игру в историю, фиксирует время и начисляет награду.
- «Дроп игры» переносит игру в историю и фиксирует время, но не начисляет награду.

## Важные файлы

- `server.js` - сервер, API и сохранение данных.
- `player.js` - логика профилей, таймера, инвентаря и истории игр.
- `treasury.js` - логика казны и поиска игр HowLongToBeat.
- `board.js` - кликабельные клетки и фишки игрового поля.
- `styles.css` - общий дизайн сайта.
- `assets/pog_rolls_map_loop.webp` - текущая карта.
- `assets/board-cell-mask.png` - точная маска кликабельных клеток карты.

## Синхронизация двух ПК

Для синхронизации кода лучше создать приватный репозиторий на GitHub. Тогда перед работой используйте `git pull`, а после изменений - `git add .`, `git commit` и `git push`.

Важно: если одновременно запускать сервер на двух ПК, у каждого будет свой `data/db.json`. Для единого баланса всех игроков нужен один общий сервер: в локальной сети или на хостинге.
